Created by georgia.chr on 14-Sep-15. This service handles the encode / decode of application's tokens (calls jsonwebtoken's methods sign and verify respectively) The user token of this application is comprised of: 1. payload which is user's id (key is 'userid'). Payload has expiration time. 2. tokenSalt declared in globals.js 3. data in a json (for example: expirationInMinute which is also declared in globals.js)

issueToken(payload) → {String}

Creates a token based on userid

Synchronous method (using jwt.sign). Create an expiration date/time for the new token. Then amend the new expiration time into the payload. Finally creates a new token that comprised of a. payload (userid), b. tokenSalt (define in globals) and c. other data like expirationInMinutes.
